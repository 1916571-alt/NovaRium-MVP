로컬 환경에서 **가장 빠르게 핵심만 구현하여 "데이터 분석가로서의 실무 역량"을 즉시 증명**할 수 있는 MVP(Minimum Viable Product) 단계의 프로젝트 구조를 제안해 드립니다.

이 MVP는 화려한 대시보드보다는 **'데이터의 흐름(Pipeline)'과 '실험의 논리(Logic)'**를 보여주는 데 집중합니다.

## 🛠️ NovaRium-MVP: 로컬 구현 아키텍처

복잡한 설정 없이 **Python 파일 1개와 SQL 파일 1개** 정도로 시작할 수 있는 구성입니다.

### 1단계: 데이터 생성 (The Generator)

* **기술:** Python + Faker
* **산출물:** `raw_data/` 폴더 내의 CSV 파일들 (`users.csv`, `orders.csv`, `ab_test_logs.csv`)
* **MVP 핵심:** 패스오더의 비즈니스 도메인을 반영한 **3개월치 가상 로그**를 생성합니다. (유저 1,000명 규모)

### 2단계: 로컬 데이터베이스 구축 (The Warehouse)

* **기술:** **DuckDB**
* **산출물:** `novarium_local.db` 파일
* **MVP 핵심:** 별도 서버 설치 없이, Python에서 `duckdb.connect('db_file')` 한 줄로 SQL 분석 환경을 구축합니다. CSV 데이터를 이 DB에 로드합니다.

### 3단계: 분석 및 실험 검증 (The Analysis)

* **기술:** SQL + Python (Pandas/SciPy)
* **산출물:** **Jupyter Notebook (.ipynb)** 또는 **단일 리포트**
* **MVP 핵심:** * **SQL:** 데이터 마트 테이블 생성 (예: 유저별 첫 주문일, 총 주문 횟수, 할당된 실험 그룹).
* **통계:** A/B 테스트 그룹 간의 전환율 차이를 구하고 p-value 계산.

## 📊 MVP로 보여줄 핵심 지표 (JD 대응)

공고에 나온 요구사항을 MVP에서 어떻게 보여줄지 매칭한 표입니다.

| 분석 항목 | 구현 내용 (MVP) | 채용공고 요구 역량 (JD) |
| --- | --- | --- |
| **코호트 분석** | 가입 주차별 재주문율(Retention) 테이블 작성 | **Retention, Cohort 이해도** |
| **세그먼트 분석** | '직장인' vs '학생' 페르소나별 주문 패턴 차이 분석 | **행동 데이터 기반 세그먼트 정의** |
| **A/B 테스트** | 푸시 알림 발송 유무에 따른 주문 전환율(CVR) 검증 | **실험 설계 및 성과 측정** |
| **지표 산출** | 전체 캠페인 비용 대비 총 매출(ROI) 계산 | **ROI, CAC 등 마케팅 지표 분석** |

## 📅 MVP 구현 로드맵 (3일 완성)

* **1일차 (데이터):** Python 스크립트로 패스오더 스타일의 가상 로그 생성 (유저, 상점, 주문).
* **2일차 (인프라/SQL):** DuckDB에 데이터 적재 후, 분석용 SQL 쿼리 작성 (Retention, RFM 산출).
* **3일차 (실험/리포트):** A/B 테스트 결과 해석 및 분석 인사이트를 담은 간단한 리포트 작성.
